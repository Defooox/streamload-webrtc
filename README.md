# streamload-webrtc
Разработать систему для загрузки и просмотра файлов через браузер с использованием WebRTC для потоковой передачи.

Техническое задание:

Сборка libwebrtc из исходников (7–12 дней)
Скачать и синхронизировать исходный код с помощью fetch и gclient
Настроить конфигурацию сборки
Выполнить сборку: Использовать gn gen и ninja для компиляции библиотеки

Настройка и запуск примера на основе собранной библиотеки (12–18 дней)
Посмотреть примеры из репозитория WebRTC
Написать/настроить сигнальный сервер
{
Реализация базового сервера для обработки подключений
Обработку сигналов для установления WebRTC-сессий
Тестирование сервера локально (curl или Postman)
}
Интеграция примера с libwebrtc(Связать собранную библиотеку с примером)

Разработка MVP функционала загрузки и синхронного просмотра (10–15 дней)
Реализация загрузки файлов: Добавить UI для выбора и загрузки видео-файлов
Интеграция WebRTC для потоковой передачи
{
Установить P2P соединение для стриминга загруженного файла(addTrack() + RTCPeerConnection)
Реализовать потоковую передачу
}
Синхронизированный просмотр:
Добавить обмен данными о позиции воспроизведения (currentTime) через WebRTC DataChannel.
Логика: При присоединении нового пользователя — синхронизировать время с текущим зрителем, пауза/плей в реальном времени.

UI/UX и обёртка (оставшееся время)
Создать простой веб-интерфейс (HTML/CSS/JS, возможно React для удобства)
Интеграция с backend Связать фронтенд с сигнальным сервером и WebRTC.
